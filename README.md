# 🎬 Kino Tj - Онлайн Кинотеатр

Современный онлайн-кинотеатр, созданный на чистом HTML, CSS и JavaScript без использования фреймворков и библиотек.

## 📋 Описание проекта

**Kino Tj** - это полнофункциональный веб-сайт онлайн-кинотеатра с современным дизайном в стиле Netflix/KinoPoisk. Проект демонстрирует возможности создания сложных интерактивных веб-приложений без использования внешних библиотек.

## ✨ Возможности

### Основной функционал:
- 🎥 **Hero-секция** с динамическим баннером фильма
- 📁 **Категории фильмов**: Популярные, Новинки, Рекомендуем
- 🔍 **Умный поиск** по названию, жанру и описанию
- 📱 **Адаптивный дизайн** для всех устройств
- 🎬 **Модальное окно** с подробной информацией о фильме
- ▶️ **HTML5 видео-плеер** с кастомными контролами

### Видео-плеер:
- Play / Pause
- Полноэкранный режим
- Регулятор громкости
- Progress bar с перемоткой
- Управление с клавиатуры (Space, Escape)
- Автоматическое скрытие контролов

### Дизайн:
- 🌙 Темная тема (Netflix-стиль)
- ✨ Плавные CSS анимации
- 🎨 Hover эффекты на карточках
- 📐 Современная типографика
- 🎭 Градиенты и тени

## 🛠️ Технологии

Проект создан исключительно с использованием:
- **HTML5** - семантическая разметка
- **CSS3** - стилизация (Flexbox, Grid, Animations, Transitions)
- **JavaScript (ES6+)** - логика приложения
- **Node.js + Express** - локальный API сервер
- **SQLite** - база данных проекта

### ❌ НЕ используется:
- React, Vue, Angular или другие фреймворки
- jQuery, Bootstrap, Tailwind
- Никаких внешних библиотек
- Никаких CDN
- Никаких сборщиков (Webpack, Vite и т.д.)

## 📦 Требования и зависимости

- **Node.js 18+** (рекомендуется LTS)
- **npm** (идёт вместе с Node.js)
- Пакеты проекта: `express`, `sqlite`, `sqlite3`

## 🤖 Telegram-одобрение заявок

Теперь заявки на покупку отправляются в Telegram-бота с кнопками:
- `✅ Одобрить`
- `❌ Отклонить`

После нажатия `Одобрить` доступ к фильму автоматически выдается пользователю.

### Настройка

1. Создайте файл `.env` на основе `.env.example`
2. Укажите:
  - `TELEGRAM_BOT_TOKEN`
  - `TELEGRAM_ADMIN_CHAT_ID`
3. Перезапустите сервер

Пример `.env`:
```env
TELEGRAM_BOT_TOKEN=123456:ABCDEF...
TELEGRAM_ADMIN_CHAT_ID=123456789
PORT=3000
```

База данных создаётся автоматически при первом запуске:
- Файл БД: `data/kino.db`

## 🚀 Как запустить

### Рекомендуемый способ (с БД SQLite)
```bash
npm install
npm start
```

После запуска:
- Сайт: `http://localhost:3000`
- Админ-панель: `http://localhost:3000/admin.html`

### Быстрый офлайн-режим (без БД, только localStorage)
Можно открыть `index.html` напрямую в браузере или через простой статический сервер. В этом режиме данные не синхронизируются через SQLite.

## 🎯 Особенности реализации

### Данные фильмов
Все данные хранятся в массиве объектов JavaScript:
```javascript
const movies = [
  {
    id: 1,
    title: "Название фильма",
    year: 2024,
    genre: "Жанр",
    rating: 8.5,
    duration: 120,
    description: "Описание...",
    poster: "путь/к/постеру.jpg",
    video: "путь/к/видео.mp4",
    category: ["popular", "new"]
  }
];
```

Реализован без сервера, работает полностью на клиенте:
- Поиск по названию

- Поиск по описанию
- Динамическое обновление результатов

### Видео-плеер
Кастомный плеер на основе HTML5 `<video>`:
- Кастомные контролы через JavaScript
- Стилизация через CSS
- Поддержка keyboard shortcuts
- Автоматическое управление видимостью контролов

## 🔧 Кастомизация

### Добавление новых фильмов
Откройте `js/main.js` и добавьте объект в массив `movies`:
```javascript
{
  id: 13,
  title: "Ваш фильм",
  year: 2024,
  genre: "Драма",
  rating: 9.0,
  duration: 120,
  description: "Описание вашего фильма",
  poster: "assets/posters/your-movie.jpg",
  video: "assets/videos/your-movie.mp4",
  category: ["new", "popular"]
}
```

### Изменение цветовой схемы
Измените CSS-переменные в `:root` в файле `css/style.css`

### Добавление видео
1. Поместите видеофайлы в папку `assets/videos/`
2. Обновите путь к видео в массиве `movies`
3. Поддерживаются форматы: MP4, WebM, OGG

## 💡 Советы по использованию

1. **Постеры**: Для лучшего вида используйте изображения в соотношении 2:3 (например, 300x450px)
2. **Видео**: Оптимизируйте видео для веба (рекомендуется H.264 кодек)
3. **Производительность**: Большие видеофайлы могут замедлить загрузку
4. **SEO**: Добавьте meta-теги в `<head>` для лучшей индексации
